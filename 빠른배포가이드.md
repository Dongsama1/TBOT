# 🚀 빠른 자동 배포 가이드

## 1단계: 자동 배포 스크립트 실행

프로젝트 폴더에서 `자동배포.bat` 파일을 더블클릭하거나 실행하세요.

```powershell
.\자동배포.bat
```

스크립트가 자동으로:
- ✅ Git 저장소 초기화
- ✅ 파일 커밋
- ✅ GitHub 저장소 연결
- ✅ 코드 업로드

---

## 2단계: GitHub 저장소 생성 (처음만)

스크립트 실행 중 GitHub 저장소가 없다면:

1. **GitHub 접속**: https://github.com
2. **로그인** (계정이 없으면 가입)
3. **새 저장소 생성**:
   - 우측 상단 **"+"** 버튼 클릭
   - **"New repository"** 선택
   - Repository name: `ai-smart-service` (또는 원하는 이름)
   - **Public** 선택
   - **"Create repository"** 클릭
4. **저장소 URL 복사** (예: `https://github.com/사용자명/ai-smart-service.git`)
5. 스크립트에 URL 입력

---

## 3단계: Render.com 배포

### 방법 A: 웹에서 배포 (권장)

1. **Render.com 접속**: https://render.com
2. **GitHub로 로그인**
3. **"New +"** 버튼 클릭 → **"Web Service"** 선택
4. **저장소 선택**: 방금 업로드한 `ai-smart-service` 선택
5. **설정 입력**:
   ```
   Name: tym-chatbot
   Region: Singapore
   Branch: main
   Root Directory: (비워두기)
   Environment: Node
   Build Command: npm install
   Start Command: npm start
   ```
6. **"Create Web Service"** 클릭
7. **배포 완료 대기** (약 5-10분)
8. **URL 확인**: 배포 완료 후 생성된 URL로 접속!

### 방법 B: render.yaml 사용 (고급)

프로젝트에 `render.yaml` 파일이 이미 포함되어 있습니다.
Render.com에서 저장소를 연결하면 자동으로 설정이 적용됩니다.

---

## ✅ 배포 확인

배포가 완료되면:

1. Render.com Dashboard에서 서비스 상태 확인
2. 생성된 URL로 접속 (예: `https://tym-chatbot.onrender.com`)
3. 챗봇이 정상 작동하는지 테스트

---

## 🔄 업데이트 배포

코드를 수정한 후:

```powershell
.\자동배포.bat
```

또는 수동으로:

```powershell
git add .
git commit -m "Update description"
git push
```

Render.com이 자동으로 재배포합니다!

## ⚠️ 중요 사항

### 진단 데이터 준비

배포 전에 `diagnostics` 폴더에 진단 파일을 넣어주세요:

```
ai-smart-service/
├── diagnostics/          ← 이 폴더에 .txt 파일 넣기
│   ├── TYM_T115_PTO.txt
│   └── ...
```

**중요**: `diagnostics` 폴더의 파일들을 Git에 커밋해야 배포됩니다!

### GitHub 인증

GitHub에 푸시할 때 인증이 필요할 수 있습니다:

**옵션 1: GitHub Desktop 사용** (가장 쉬움)
- https://desktop.github.com 다운로드
- GitHub 계정으로 로그인
- 저장소 클론 후 사용

**옵션 2: Personal Access Token**
- GitHub → Settings → Developer settings → Personal access tokens
- "Generate new token" 클릭
- `repo` 권한 선택
- 토큰 생성 후 비밀번호 대신 사용

## 🆘 문제 해결

### "git 명령어를 찾을 수 없습니다"
- Git 설치 확인: https://git-scm.com/download/win
- PowerShell 재시작

### "GitHub 인증 실패"
- GitHub Desktop 사용 권장
- 또는 Personal Access Token 생성

### "배포 실패"
- Render.com 로그 확인
- `package.json`의 `engines` 필드 확인
- Node.js 버전 확인
