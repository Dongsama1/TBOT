# 🚀 무료 배포 빠른 시작 가이드

## 1단계: GitHub에 코드 업로드

### Git 초기화 (처음 한 번만)
```bash
git init
git add .
git commit -m "Initial commit"
```

### GitHub 저장소 생성 및 업로드
1. https://github.com 접속 후 로그인
2. 우측 상단 "+" 버튼 클릭 → "New repository"
3. 저장소 이름 입력 (예: `ai-smart-service`)
4. "Create repository" 클릭
5. 아래 명령어 실행:

```bash
git branch -M main
git remote add origin https://github.com/사용자명/ai-smart-service.git
git push -u origin main
```

---

## 2단계: 진단 데이터 준비

프로젝트 폴더에 `diagnostics` 폴더를 생성하고 진단 파일(.txt)을 넣어주세요.

```
ai-smart-service/
├── diagnostics/          ← 이 폴더 생성
│   ├── TYM_T115_PTO.txt
│   ├── TYM_K110E_DEF.txt
│   └── ...
├── server.js
├── package.json
└── ...
```

**중요**: `diagnostics` 폴더의 파일들을 Git에 커밋해야 배포됩니다!

```bash
git add diagnostics/
git commit -m "Add diagnostics data"
git push
```

---

## 3단계: Render.com에 배포

### 1. Render.com 가입
- https://render.com 접속
- "Get Started for Free" 클릭
- GitHub 계정으로 로그인

### 2. 새 서비스 생성
1. Dashboard에서 "New +" 버튼 클릭
2. "Web Service" 선택
3. GitHub 저장소 연결 (처음이면 권한 승인)
4. 저장소 선택: `ai-smart-service`

### 3. 서비스 설정 입력
- **Name**: `tym-chatbot` (원하는 이름)
- **Region**: `Singapore` 선택
- **Branch**: `main`
- **Root Directory**: (비워두기)
- **Environment**: `Node`
- **Build Command**: `npm install`
- **Start Command**: `npm start`

### 4. 배포 시작
- "Create Web Service" 클릭
- 배포가 자동으로 시작됩니다 (약 5-10분 소요)

### 5. 배포 완료
- 배포가 완료되면 "Your service is live" 메시지가 표시됩니다
- URL이 생성됩니다: `https://tym-chatbot.onrender.com`
- 이 URL을 복사하여 브라우저에서 접속하세요!

## ⚠️ 주의사항

1. **무료 티어 제한**
   - 15분간 사용하지 않으면 서비스가 슬립 모드로 전환됩니다
   - 첫 요청 시 약 30초 정도 시작 시간이 필요합니다

2. **진단 데이터**
   - `diagnostics` 폴더의 파일은 반드시 Git에 커밋되어야 합니다
   - `.gitignore`에서 `diagnostics/`가 제외되어 있지 않은지 확인하세요

3. **데이터베이스**
   - SQLite 데이터베이스는 재배포 시 초기화될 수 있습니다
   - 중요한 데이터는 별도로 백업하세요

## 🆘 문제 해결

### 배포가 실패하는 경우
1. Render.com Dashboard에서 "Logs" 탭 확인
2. 오류 메시지 확인
3. 일반적인 문제:
   - Node.js 버전 문제 → `package.json`의 `engines` 필드 확인
   - 의존성 설치 실패 → `package.json` 확인

### 서비스가 작동하지 않는 경우
1. "Logs" 탭에서 서버 로그 확인
2. 진단 데이터가 로드되었는지 확인
3. `/api/refresh` 엔드포인트 테스트
